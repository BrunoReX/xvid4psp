AAD профиля

  • AAD разработан специально для аниме IVTC и его деинтерлейса с помощью данного скрипта Avisynth.
  • В IVTC, AAD использует дополнительный режим обработки 4pass, с оригинальным AADivtc сценарием, обеспечивает дополнительную поддержку для автоматической обработки одновременно поддерживая руководство написанного TIVTC OVR-файлa. Градиент сцена может чистым и красивым 3rdpass +4 thpass чертежи уничтожены, вам больше не нужно вручную настроить градиент частично разнесенных остатка. Несмотря на то, AAD в 3pass предельный режим медленной скорости, но на экране чертеж, чтобы справиться с экстремальными Bad Edit особенно эффективным.

  • Из поблагодарить Avisynth разработчиков и производителей DLL плагина есть друзья , спасибо NMM форум поможет мне сделать этот скрипт и вдохновения!

Параметр Описание

первой части, главной функцией: AAD () таблицу параметров

·         mode(режим) = 0/1/2 (целое число, значение по умолчанию: 1)
AAD режим обработки:

  •  0 - выход TFM файл журнала будет создаваться в том же каталоге, источник в фильме файл с именем tfm_log.txt. Использование этой опции войти только выход, весь фильм используется независимо mode = 0 вариант.
  •  1 - дополнительно-4pass из IVTC режим обработки, я рекомендую эту опцию в IVTC.

       2 -  Дополнительный 3pass проходный режим обработки IVTC, оригинальный сценарий: AADivtc.


* AADivtc, в теории, это мой оригинальный алгоритм. Прежде всего, это моя собственная идея, во-вторых, я не видел никого эту операцию. Но на самом деле с IsCombedTIVTC () и tdecimate () функцияvb, представленными двумя внешними плагинами удалось добиться, это не считается полном смысле этого оригинала. ADivtc службы в AAD (mode = 2), хотя я должен создать отдельную функцию, но, пожалуйста, не используйте его самостоятельно. Это очень простой алгоритм, но если вам повезет, она может достигать 100% правильный матч, не будет рывка явление. Но только в идеальной ситуации, в реальной эксплуатации, это в основном только поддерживать скорость более чем на 60% правильно. В AviSynth нет возможности использовать более продвинутый алгоритм, так что AADivtc по-прежнему прост, но иногда это может быть более совершенным, чтобы решить некоторые неразрешимыми щеткой. При установке режима mode = 2, будет AADivtc () для выполнения 1-й проход, 2-й проход будет вынуждена закрыться, ovr больше не в силе, 3-й проход четвёртый проходы все еще ​​могут быть. Пожалуйста, обратите внимание, если у Вас есть такая установка: AAD (mode = 2, ivtc3pass = false, finalpass =false) , это тоже самое, как и в AADivtc () полос, я настоятельно рекомендую вам сделать это. Вы можете наблюдать режим mode = 2 обрабатывать фильм не найдете то, что это неправильно, но я предлагаю вам не использовать весь исходник в режиме mode = 2 варианта. Режим mode = 2 разработан с mode = 1 дополняют друг друга, если вы используете режим mode = 1, короткий вас не устраивает, вы можете попробовать режим mode = 2, если вы считаете, что лучше, вы можете использовать часть заменить кадр, который вы не удовлетворены кадра. В AADTrim набор функций может легко это сделать, следуя  указаниям скрипта, показуя, как просто режим обработки кадров c 10-20 и c 30-40 замены кадров для работы в режиме mode=2 рассматривается.


mode2=last.AAD(mode=2)

AAD (mode = 1). AADTrim (10, 20, clip1 = mode2). AADTrim (30, 40, clip1 = mode2)

 

TFF = True/False (логическое значение по умолчанию: True)
True, когда верхнее поле уидёт первым (TFF), false когда нижнее поле первое (BFF).
※ Примечание: Boolean параметр установлен в истинное (TRUE), открытые, ложный (ложная), чтобы выключить, то же ниже.

·         debug = true／false (логический, значение по умолчанию: FALSE)
режиме отладки. Открытое будут отображаться на экране МТФ и AAD обработки информации, и выход TFM отладочной информации (доступно программное обеспечение DebugView для просмотра).
Обратите внимание, если вы просто хотите таблетки, не открывайте эту опцию, иначе вы давление хороший фильм будет отмечен с субтитрами.

·         finalpass = True / False (логический, значение по умолчанию: True)
последнего прохода операции, убить остаточные шахматном, шатаясь, остаточная сцене градиент является особенно эффективным в то же время, не погубить прогрессивного кадра. Эта операция происходит автоматически, Вам не нужно вручную Trim (начало, конец), чтобы очистить градиент сцене в шахматном порядке. Конечной = False эффективным. Однако, если предыдущие шаги остаточной лица в шахматном порядке, finalpass не может быть в состоянии решить вашу проблему. Этот параметр рекомендуется открывать.

·         ovr = "X: \ хх \ XX.txt" (строка Значение по умолчанию: "noovr)
ovr- файл, указанный TFM эффективно, когда режим mode= 1. Когда ovr будет указано, TFM в любом случае будет помещен в положение 1-й прохода, хотя внутри ovr ничего не пишется. Если вы не укажете ovr, то a60224 займёт позицию по умолчанию на 1-м проходе. Вы можете установить ovr = "noovr". В этом случае OVR, временно закрыты в целях облегчения отладки. Когда режим mode= 2, независимо от того, какой вы указываете OVR, он не вступит в силу. Формулировка OVR см. руководство TIVTC, NMM форуме этот пост (: dgwxx), в то же время, чтобы быть полезными для Вас.

·         ivtc2pass = True / False (логический, значение по умолчанию: True)
IVTC операции во втором проходе. Когда режим mode= 1. Рекомендуемая открытым.

·         ivtc3pass = True / False (логический, значение по умолчанию: True)
IVTC операцию третий проход. Когда режим = 1/2. Рекомендуемая открытым.

·         ultimate = True / False (логический, значение по умолчанию: FALSE)
IVTC области предельный режим для полного кадра при обнаружении движения альфа-смешения, когда ivtc3thpass = True отменяет finalpass. Пожалуйста, обратите внимание, что только шахматном картина движения в прозрачных гибридных поле ореолы лечения для правильного ореолов. Преимущество конечной скоростью Deinterlace режиме может достигать 100%, недостатком является то, что экран будет иметь некоторые расплывчатые и очень медленное кодирование займет больше времени.

Вторая часть, дополнительные независимые функции: AADTrim () таблицу параметров

·         STF / EDF (целое число, значение по умолчанию: нет)
начала кадра / конца кадра.
Почти в точности равна синтаксис Trim (). Но AADTrim (0,0) представлены равен 0, а не из кадра 0 до последнего, который является AADTrim и отделка только разница в цифрах. Потому что вы не должны использовать эту функцию для всех операционных кадров приложения функции для фильма. Обратите внимание, AADTrim не очень усеченном кино, но фильм обработки сегмента соединены вместе. STF / EDF обязательный параметр должен быть заполнен.

·         func / func2 (строка, значение по умолчанию: нет)
пользовательские функции действия.
Параметр функции func определяет работу функции будет применяться к части из STF для EDF, func2 является параметр определяет работу функции будут применяться к STF EDF за пределами части. Вы должны убедиться, что после себя определенные функции после операции, частота кадров (кадров в секунду), разрешение кадра (х и у), а также цветовое пространство, а другая часть, которая должна быть подключена, иначе фильм не может быть соединены последовательно. Функция внутри пользовательских действий может включать только функции, а не другие вещи, как func2 = "last.warpsharp ()" не является правильной формулировки. функции и func2 параметры являются необязательными, и вы не можете заполнить.

·         clip1 / clip2 (фильмов, клипов, значение по умолчанию: нет)
видеоклипов на замену.
Запасные части от STF к клипам EDF фильма clip1 параметров, введенных;
клипа на clip2 параметров, введенных из STF для EDF , чем другие части будут заменены .
Если указать параметр функции clip1 заменит часть ФРЭ от STF, а затем заменить в конце этой части параметров приложения функции указанных функций;
если Вы укажете func2 параметров, из clip2 заменяет от STF для EDF за пределами части, а затем по этой части, чтобы заменить готовых применять функцию func2, заданное параметром.
clip1 clip2 являются необязательными параметрами не может заполнить.

Часть III, дополнительные независимые функции: AUS720 () таблицу параметров

·         a / w / d / s (целое число, значение по умолчанию: 0)
это собственно кроп слева, сверху, справа и снизу.
В соответствии с клавиатурой  a、w、d、s направление, представляющее слева, сверху, справа, под обрезку количество пикселей и изменение размера фильтра внутренней отделки, эти цифры эквивалентны отсечения до укрупняются.
AUS720 (8,0,8,0) может быть эквивалентно LanczosResize (1280,720,8,0, -8,0).

AAD случае использования

  • IVTC:
AAD ()
  • Выходной файл TFM журнала:
AAD (mode= 0)
  • Использование OVR TFM:
AAD (ovr = "X: \ хх \ XX.txt")
  • Режим отладки:
AAD (DEBUG = True)
  • Используйте AADTrim различных фильтров резкости под различные фрагменты из фильма:
AADTrim (100,200, "WarpSharp ()", "UnsharpMask ()")
  • AUS720 проста в использовании, например:
AUS720 (8,0,8,0)
