HybridFuPP version 0.991b



INTRODUCTION

This Avisynth script allows both resizing and filtering of your videos. Distinct filters can be applied on different parts of a same picture (edges, static areas, motion, dark and bright areas, skin...) : HybridFuPP is a kind of adaptive processor, allowing picture cleaning, general perception improvements and compressibility gain.

Main idea consists in filtering strongly the less visible parts of the picture, and very softly the more visible. HybridFuPP uses masks to detect areas to process.

In most recent versions, HybridFuPP proposes a more aggressive “psychovisual” approach (although very empirical), allowing to “force” codecs, used while re-encoding output clip, to allocate more bits where and when it is considered to be critical, especially at low bitrates (dark scenes, skinned areas...). All of these new features are of course optional.

Standard processings proposed are : resizing (8 possibilities), denoising (optionally motion-compensated), dark and bright areas denoising and / or enhancing, edges sharpening or softening (distinct horizontal and vertical settings), edges brightness control, deringing (removing artefacts around edges), deblocking (removing blocks effect generated by mpeg compression) and skin processing.

Although HybridFuPP has been originally designed to process rather clean materials, filtering package used is flexible and powerful enough to be used with more noisy materials too (depending on noise type) : increasing default settings could be though required.

HybridFuPP is provided with some presets allowing beginners to use the script easily and other people to use these presets as basis for their explorations. Moreover, HybridFuPP allows users to define their own processing chains : lot of things become then possible !



PREREQUISITES

HybridFuPP requires Avisynth v 2.58.

Put HybridFuPP.avsi and provided filters in your avisynth plug-ins directory (by default : C:\Program Files\AviSynth 2.5\plugins). FFT3DFilter.dll requires an additional dll (FFTW3.dll) that must placed into you windows\system32.

NB : these DLLs are included in some compressed files, you need first to extract them from each archive

Be sure that UnFilter.dll (v1.5), Deen.dll (v1.0 beta 1), FFT3DFilter.dll (2.1.1), MVTools2.dll (2.4.2), MT_Masktools.dll (v2.0.36), Deblock (v1.2) and Addgrain (v1.0) are loaded in your main script.

C A U T I O N : You MUST use filters versions provided with HybridFuPP, or you will probably get unexpected results or errors !

Denoising :

Use these parameters to clean your clip and/or to gain compressibility.

HybridFuPP can natively call 3 different denoisers : deen, mdegrain or Fft3DFilter. It is possible to switch between them using 'D_Mode' parameter. Some parameters are common for all of them, some others are specific to one mode

Mdegrain (D_Mode = 2) is natively a motion-compensated denoiser. But what is really new from HybridFuPP 0.99a is the ability to use motion compensation whatever denoiser you decide to use. It is of course true when using Deen or Fft3DFilter (D_Mode=1 or D_Mode=3). But it is even possible to use any temporal or spatio-temporal denoiser with HybridFuPP, using N1 or N2 parameters (see section 'Advanced Parameters'), and get it motion-compensated ; you will have then to set motion compensation parameters, ie M_TRadius and Bl_Size for the most.

Temporal radius, ie number of consecutive frames taken into account for motion computation, has to be set using parameter M_TRadius

M_TRadius = 0 means no motion compensation (not allowed when D_Mode=2)

M_TRadius set between 1 and 3 indicates the number of frames before and after the current frame that must be taken into account for motion compensation computation

NB : if you want to use an external denoiser (using N1 or N2 parameters) which is internally already motion compensated, you should then deactivate HybridFuPP's motion compensation to avoid double work (and maybe unexpected results). Use M_TRadius = 0 for that.

Caution : Motion compensation is very time consuming. The higher the temporal radius, the higher the processing time !



See below all denoising parameters available :

D_Mode

1..3

Denoiser selection :

1 : Deen (default for anime presets)

2 : Mdegrain[1-3] (default in most presets)

3 : FFT3DFilter (default for 'Very High' preset)



S_Str

0..100

Spatial filtering strength (used when D_Mode = 1 or D_Mode = 3)

S_Radius

0..3

Radius for spatial filtering (only used when D_Mode = 1)

S_Dist

0.0 ..1.0

Parameter influencing spatial filtering threshold. The weaker, the more edges are preserved (only used when D_Mode = 1)

T_Str

0..100

Temporal filtering strength (only used when D_Mode = 1)

C_Str

0..100

chroma filtering (only used when D_Mode = 1)

MP_Mode

0..2

Motion processing mode :

0 : no motion processing

1 : motion-compensated temporal or spatio-temporal denoising (default)

2 : apply only spatial denoising to motion ('motion blur')

M_TRadius

0..3

Temporal Radius for motion compensation (ie number of frames before and after a given frames to take into account for motion compensation computation). 0 means no motion compensation. M_TRadius can not be set to 0 when using Mdegrain (D_Mode = 2). M_TRadius is automatically set to 1 when using MP_Mode = 2 (motion blur)

Bl_Size

4, 8 or 16

Block size for motion estimation

ME_Resizer

0..8

Resizer used for super-clip transformation (used for motion estimation in all cases, but also directly used by Mdegrain for denoising).

This parameter can be useful when using Mdegrain denoiser (D_Mode = 2) to gain more details. It should be let to its default value (0) in all other cases. Change it only if you understand what you do. Resizer values are those described in section 'General Parameters'

M_Thr

0..255

Motion detection threshold

M_SCD

0..999

Scene change detection threshold

M_Str

0..300

Spatial denoising strength (used only when MP_Mode = 2)

if you don't like washy faces on your encodes, this is the way to go,,,

ADVANCED PARAMETERS

HybridFuPP allows you to define your own filtering chains, after the resizer for dark and bright areas processing, before or after the resizer for edges and denoising.

Resizing remains processed by the main script (but can be defined using the Resizer parameter)

N1

filtering chain (denoisning), applied before resizing

N2

filtering chain (denoising), applied after resizing

E1

filtering chain for edges, applied before resizing

E2

filtering chain for edges, applied after resizing

M

filtering chain on motion (only used for optional motion blur, ie when MP_Mode is set to 2)

LP

filtering chain for dark and bright areas

DR

filtering chain for deringing



NB : filtering chains have to be defined using quotes

Ex : HybridFuPP(640, 480, preset = "medium", N2 = "deen()" )

if the filtering chain contains quotes, then it has to be defined using triple quotes

ex : HybridFuPP(640, 480, preset = "medium", N2 = "deen()" , Dering = true, DR = """BlindPP(quant= 2, cpu2 = "xxxxxx", moderate_h = 40, moderate_v = 80)""")



All advanced parameters which are not explicitly defined are then replaced by HybridFuPP default processing chains (see further).

It is therefore possible to re-define only some of the advanced parameters (ex : you can define only N1, N2 and LP ; other advanced parameters, i.e. M, E1, E2 and DR, will use default HybridFuPP’s processing chains).

To cancel a default processing chain without defining a new one, you have to use double quotes, and write XX = "", where XX is an advanced parameter (ex : E2 = "" )

Caution : it is not possible to add a filter to a default chain ; the whole chain must be entirely redefined.

EXAMPLES

HybridFuPP( 448, 320 )

448x320, use default values. Is therefore equivalent to HybridFuPP(448, 320, preset = "high")

HybridFuPP( 448, 320, preset="low" )

448x320, use low quality preset

HybridFuPP( 448, 320, preset="medium", E_Str_X= 40, E_Str_Y = 10 )

448x320, preset medium, but applying oversharpening (using a stronger value on horizontal axis, in order to get better aspect for SVCDs for instance)

HybridFuPP(preset = "very high", D_Mode = 1, S_Str = 8, T_Str = 5, C_Str = 0 )

448x320, using preset "very high", except for denoising : Denoiser Mode has been set to 1 (using deen), S_Str (spatial filtering strength) and T_Str (temporal filtering strength) have been set respectively to 8 and 5. Chroma channels will not be processed

HybridFuPP( 448, 320, MP_Mode = 2, M_Str = 160, dering = true )

448x320, uses default values (see preset  "high"), except for M_Str (motion filtering strength) and dering

HybridFuPP( 448, 320, MP_Mode = 2, M_Str = 160, dering = true, show = "M" )

Same than the previous one, but motion mask is displayed

HybridFuPP( 448, 320, preset = "very high", LP = "" )

448x320, using preset “very high”. Dark and bright areas processing is though disactivated.

HybridFuPP( 448, 320, show = "MDB", B_Thr = 165 )

448x320, displays motion (M), dark areas (D) and bright areas (B). Bright areas detection threshold is set to 165

HybridFuPP( 448, 320, N2 = "temporalsoften(2,3,3,6,2)", M_TRadius = 2, deblock = true)

448x320. HybridFuPP's default N2 processing is replaced with temporalsoften, using 2 frames before and after the current one for motion compensation. Deblocking has been activated.
