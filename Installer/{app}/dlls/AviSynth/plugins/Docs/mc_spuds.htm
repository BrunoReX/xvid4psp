Description

This script will provide a wide array of denoising videos utilizing motion compensated (mvtools) methods and mask (masktools).   To simplify usage a named parameter strength will set many defaults so you don't have to.  Of course they are all override-able should you wish to fine tune and play. 

Required Functions

There are many tools needed to run the script, using them all correctly and in conjunction with one another is part of what the script does for you.  Not all of these functions are used every time, some are unique to various special modes that you can apply to your video, so if you want to utilize everything the script offers then you should get all of these tools.

What the script does:

  • Preprocess (optional) for motion vector detection (This step is used for improved improved motion search, not for final usage).  This can provide some very powerful results.  By cleaning the clip before motion detection the MV engine can focus on real motion and not be distracted by noise / grain dancing.  The pre processed clip is very calm and only used by the motion search. 

  • Get motion vectors with MVAnalyse

  • For frames = 1, 2, 3 or 4 use the motion vectors to MVDegrain

  • For frames = 1,2,3 or 4 with flow=true option use the vectors to create a long temporal clip with MV Compensate.  This clip is then denoised and the key (center) frame returned.  A much different noise removal then MVDegrain and can provide very good results but can also cause some oil painting.  The denoiser is ff3dfilter or dfttest depending on the number of frames being used.

  • Aggressive (optional) .  This will cause two passes through the function, the first pass is mild while the second pass is as defined by strength or the parameters passed to the function.

  • Temporal (optional).  This is also a two pass through the degraining function but with a special limiting clamp to avoid removing to much detail.  Good for dancing grain.

  • Next it will create a motion mask and static mask  based on the now degrained / denoised frame

  • Apply sharpening (either contra, limitedsharpenfaster, motion compensated or none) to the static mask (ie sharpen only what is not moving).  With contra and motion compensated it makes sure it does not add back more than what it removed.

  • Compute a conservative edgemask and dering those edges to remove artifacts and noise that appear on edges

  • Optional post process if you really have a bad clip.  With post it is limited in how much it can change since at this point the clip been worked on quite a bit.

  • That is the basic flow, but there are many additional options as listed below

Starfield Protection
starfield true,false false Creates a mask of the frames white spots and re inserts them into the denoised frame IF the luma value of the created mask is above the thStar threshold.  If real noise is getting through increase thStar or if stars are still being removed decrease thStar.  Its very sensitive and can cause noise to remain since to a filter a spot is a spot.
lumathres int 150 Cut off value for luma.  All  identified spots that are above this luma value will be added to the star mask.  The total luma value of this  mask is validated against thStar which prevents a mask with a few spots from being accepted as valid.  Do not go below 130 for this value, lower values will all more stars to be included.
thStar float 0.0925 Luma threshold for keeping a star mask or discarding it as noise.
Anime / Toon processing
anime true,false false Sets various processing parameters that are good for anime / toon sources ie keeping lines intact, scene change detection, etc.  Also turns on a artifact prevention routine for larger blocksizes.
Adding Grain
addnoise true,false false Adds random grain to dark areas of the frame, this can help prevent blocking when the file is encoded.  Grain is added to areas with luma below shadow_l and does a linear transition from adding grain to no grain adding between shadow_l and shadow_h.  Above shadow_h no grain is added.
shadow_l int 24 low limit for add noise, luma below this value will get grain added
shadow_h int 42 high limit for add noise, luma above this value is untouched. 
Focus
focus true/false false May or may not help provide a crisp edge where a blurry one exists
Aggressive
aggressive true/false true Does a two pass denoising.  The first pass is a lite first pass and the second a normal (as specified) noise removal. The style of denoising will be the same for both passes (ie mvdegrain or mvcompensate) Slow but aggressive.

ss_y float 1.25 Passed to limited sharpen faster, this value is a smode=4 value

MVFlow MVCompensate Processing
flow true/false false Used  to determine if it will use mvcompensated or mvdegrained noise removal.  Each have their strengths and weakness so see what you like.  Frames=4 is always flow=true.  Flow will create a long temporal clip that it denoises and then it will pull out the cleaned key frame and discard the wing frames.
fs1,fs2,fs3,fs4 float 3,4,3,2 Sigma values for denosing the long temporal clip.  Note the temporal clip is already been denoised with mvcompensate!
Post Processing
postprocess true/false false Option to run a second cleaning operation on the clip.  Can be useful if the clip is very noisy.  The amount of change is limited to help prevent over smoothing of the image
Pre Processing
prefilter 0,1,2,3,4 2 Easy way to set preprocessing options.  0=None, 1=Fast, 2=Normal strength based, 3=Max, 4=Motion compensated
preprocess true/false strength Default based on strength value, can override the default by passing the prepreocess=true/false.  Preprocess cleans the input clip so that MVAnalyse can do a better job finding motion and not getting distracted by noise.  If the clip is very clean you can set this to false, otherwise leave it true.

prefast true/false false fast mode preprocess function.  Increases speed of function but will not clean as well.
premc true/false true Apply prefiltering with a motion mask to avoid removing details in the moving areas while still calming the static areas. Set on with prefilter=2 or 4
Temporal
temporal true/false false Turns on temporal processing which can help with videos that have 'dancing' or very heavy grain.  Its a special limiting function that prevents mvtools from removing more from an area then what a very strong non motion compensated denoising would.  Helps remove dancing grain.
